১. DOM এলিমেন্ট সিলেক্টরদের পার্থক্য
getElementById: এটি শুধু একটি নির্দিষ্ট ID দিয়ে এলিমেন্ট খুঁজে বের করে। যেহেতু একটি পেজে একটি ID একবারই থাকতে পারে, এটি খুব দ্রুত কাজ করে। (যেমন: document.getElementById('total-count'))

getElementsByClassName: এটি একটি ক্লাসের সব এলিমেন্টকে একটি লিস্ট (HTMLCollection) আকারে দেয়। এটি "লাইভ", অর্থাৎ পরে কোনো এলিমেন্ট যোগ করলে এটি আপডেট হয়।

querySelector: এটি সবচেয়ে শক্তিশালী। আপনি CSS-এর মতো করে এলিমেন্ট ধরতে পারেন (যেমন: .class, #id, বা div > p)। এটি শুধু প্রথম এলিমেন্টটি দেয়।

querySelectorAll: এটি CSS সিলেক্টর দিয়ে সব এলিমেন্ট খুঁজে বের করে একটি স্থির লিস্ট (NodeList) দেয়।

২. DOM-এ নতুন এলিমেন্ট তৈরি ও যোগ করা
জাভাস্ক্রিপ্ট দিয়ে ডাইনামিকালি কিছু করতে হলে ৩টি ধাপ ফলো করতে হয়:

তৈরি করা: document.createElement('div') দিয়ে একটি মেমোরিতে এলিমেন্ট বানানো হয়।

সাজানো: সেই এলিমেন্টের ভেতরে টেক্সট বা ক্লাস যোগ করা (যেমন: element.classList.add('card'))।

যোগ করা: appendChild বা prepend দিয়ে সেটিকে মেইন HTML-এর কোনো একটা ডিভ-এর ভেতরে ঢুকিয়ে দেওয়া। আপনার কোডে আমি jobsContainer.appendChild(card) লিখেছি, যা এই কাজটিই করে।

৩. Event Bubbling (ইভেন্ট বাবলিং)
এটি জাভাস্ক্রিপ্টের একটি মজার ধর্ম। যখন আপনি কোনো বাটনে ক্লিক করেন, সেই ক্লিকটি শুধু বাটনেই সীমাবদ্ধ থাকে না। এটি বাবল বা বুদবুদের মতো উপরে উঠতে থাকে। প্রথমে বাটন, তারপর তার বাবা (Parent Div), তারপর পুরো বডি—সবাইকে সে জানায় যে "এখানে একটা ক্লিক হয়েছে"।

৪. Event Delegation (ইভেন্ট ডেলিগেশন)
বাবলিং-এর সুবিধাকে কাজে লাগিয়ে আমরা ইভেন্ট ডেলিগেশন করি। ধরুন আপনার প্রজেক্টে অনেকগুলো জব কার্ড আছে। প্রতিটি কার্ডের বাটনে আলাদা করে addEventListener না দিয়ে, আপনি যদি তাদের কমন "প্যারেন্ট ডিভ"-এ একটি লিসেনার দেন, তবে সে-ই সব বাচ্চার ক্লিক হ্যান্ডেল করতে পারবে।

সুবিধা: এতে মেমোরি কম খরচ হয় এবং পরে নতুন কোনো কার্ড যোগ করলেও সেটি অটোমেটিক কাজ করে।

৫. preventDefault() বনাম stopPropagation()
এই দুটি প্রায়ই আমাদের কনফিউজ করে দেয়:

preventDefault(): এটি এলিমেন্টের নিজস্ব স্বভাব বন্ধ করে। যেমন: একটি লিংকে ক্লিক করলে অন্য পেজে চলে যাওয়ার কথা, কিন্তু এটি ব্যবহার করলে সে আর যাবে না।

stopPropagation(): এটি ইভেন্টকে উপরে উঠতে (বাবলিং হতে) বাধা দেয়। ধরুন আপনি বাটনে ক্লিক করেছেন কিন্তু চান না যে বাটনের বাবা (Parent) জানুক যে ক্লিক হয়েছে, তখন এটি ব্যবহার করবেন।
